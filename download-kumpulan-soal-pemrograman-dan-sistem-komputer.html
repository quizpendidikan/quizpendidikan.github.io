Jawaban

Robot merupakan sebuah prototype yang dapat dikontrol secara otomatis untuk membantu

pekerjaan manusia. Beberapa unsur utama dalam membuat sistem robot adalah membangun

peralatan mekanik, perangkat keras dan perangkat lunak. Keterbatasan biaya dalam membuat

perangkat keras robot dan kemampuan pemrograman merupakan hambatan utama dalam

mempelajari dan mengembangkan robot. Berkembangnya komunitas robot pada saat ini menjadikan

Robot Operating System (ROS) dan Gazebo sebagai media interaktif untuk belajar dan membuat

simulasi robot. ROS merupakan sebuah middleware yang dilengkapi dengan tools dan library untuk

membuat perangkat lunak robot, sedangkan Gazebo merupakan sebuah aplikasi simulasi 3D yang

dapat digunakan untuk membangun desain perangkat keras robot. Dalam penulisan ini, peneliti

membuat sebuah simulasi robot penghindar benda menggunakan sensor lidar dan kamera, simulasi

robot ini dibangun menggunakan ROS dan Gazebo dan dapat digunakan sebagai media

pembelajaran robot interaktif.

Kata Kunci: Robot Operating System (ROS), Gazebo, Robot.

Abstract

A robot is a prototype that is able to control by automatically to help the human work. Some of

the parts to build a robot system are making the mechanical system, hardware, and software. The

limitation of cost to make the robot hardware and programming skills are an obstacle to learn and

develop the robot system. The development of the robot community nowadays makes the Robot

Operating System (ROS) and Gazebo as an interactive media to learn and make the robot simulation.

ROS is a middleware that equipped with tools and libraries to create a robot software, while Gazebo is

a 3D simulation application to build a robot hardware design. On this paper, the researcher makes a

robot simulation to avoid the object using Lidar sensor and camera, the simulation has developed

using ROS and Gazebo that be able to use as a media for robot learning interactive.

Keyword: Robot Operating System (ROS), Gazebo, Robot.

Bahasa rakitan digunakan untuk memanipulasi perangkat keras secara langsung, mengakses instruksi CPU khusus, atau mengevaluasi masalah kinerja. penting Bahasa assembly juga digunakan untuk meningkatkan keunggulan kecepatannya dalam pemrograman komputer. 8. Apa perbedaan antara Berpikir Komputasional dan pemrograman komputer? Berikan contoh perbedaan antara Berpikir Komputasional dan pemrograman komputer!​


Jawaban:

Berpikir Komputasional (Computational Thinking) dan pemrograman komputer adalah dua konsep yang saling terkait dalam bidang ilmu komputer, namun memiliki perbedaan dalam fokusnya.

Berpikir Komputasional adalah suatu kemampuan dalam memecahkan masalah yang mendasarkan pada pemikiran komputer. Hal ini melibatkan pemecahan masalah secara logis dengan menggunakan prinsip-prinsip komputasi seperti pemecahan masalah secara algoritma, pengenalan pola, dekomposisi masalah, dan abstraksi. Berpikir komputasional lebih berfokus pada pemecahan masalah secara umum bukan tergantung pada suatu bahasa pemrograman tertentu. Contohnya adalah ketika kita menghadapi suatu masalah kompleks, berpikir komputasional akan membantu kita untuk memecahkannya menjadi langkah-langkah yang lebih sederhana dan menjalankannya secara terurut.

Pemrograman komputer adalah proses membuat program melalui penulisan kode untuk menciptakan suatu aplikasi atau perangkat lunak. Pemrograman komputer melibatkan penggunaan suatu bahasa pemrograman tertentu dan sintaks yang tepat untuk menginstruksikan komputer dalam menjalankan suatu tugas atau perintah. Contohnya adalah ketika kita ingin membuat suatu program untuk menghitung luas segitiga, kita akan menggunakan bahasa pemrograman seperti Python untuk menulis kode yang menginstruksikan komputer melalui serangkaian perintah matematika untuk menghitung luas segitiga tersebut.

Jadi, perbedaan antara Berpikir Komputasional dan Pemrograman Komputer dapat diringkas sebagai berikut:

1. Berpikir komputasional lebih merupakan pendekatan dan keterampilan umum dalam memecahkan masalah dengan menggunakan prinsip komputasi, sedangkan pemrograman komputer lebih merupakan implementasi konkret dari berpikir komputasional dalam bentuk kode program.

2. Berpikir komputasional melibatkan memecahkan masalah secara logis dan sistematis tanpa terikat pada bahasa pemrograman tertentu, sedangkan pemrograman komputer melibatkan penulisan kode menggunakan bahasa pemrograman tertentu.

3. Berpikir komputasional berfokus pada pemecahan masalah secara umum, sementara pemrograman komputer berfokus pada pemecahan masalah khusus menggunakan bahasa pemrograman.

24. Apa yang dimaksud dengan bahasa pemrograman komputer


Penjelasan:

Bahasa pemrograman komputer adalah sebuah bahasa yang digunakan untuk menulis program komputer, yaitu serangkaian instruksi yang akan dieksekusi oleh komputer untuk menyelesaikan suatu tugas atau memecahkan suatu masalah. Bahasa pemrograman memungkinkan programmer untuk menulis kode yang lebih mudah dipahami oleh manusia dan kemudian dikonversi ke dalam kode yang dapat dimengerti oleh mesin komputer.

Bahasa pemrograman komputer terdiri dari sintaks dan struktur yang ditetapkan dengan ketat, serta aturan dan konvensi tertentu yang harus diikuti untuk menulis kode yang valid. Terdapat berbagai macam bahasa pemrograman, seperti Python, Java, C++, Ruby, dan lain-lain, yang masing-masing memiliki karakteristik, kegunaan, dan keunggulan yang berbeda-beda. Program yang ditulis dalam bahasa pemrograman dapat digunakan untuk berbagai macam tujuan, seperti membuat aplikasi, website, game, sistem operasi, dan lain-lain.

40. sebutkan langkah-langkah dalam pemrograman komputer​


Jawab : Langkah langkah tersebut ialah Problem Definition,Problem Analys, Algorithm Development,Coding and Documentation,Testing , dan Maintenance

1. Problem Definiton

Problem Definition ialah menjelaskan dan memahami suatu masalah hingga ke akar-akarnya. Kita harus memahami masalah intinya,kebutuhan kita, dan output yang akan kita hasilkan

2. Problem Analys

Problem Analys ialah menganalisis suatu masalah dengan menentukan apa saja yang akan kita gunakan seperti variable,fungsi, dll.

Di Fase ini kita juga memikirkan bagaimana solusi dari permasalahan terebut

3. Algorithm Development

Di fase ini kita menentukan algoritma dalam penyelesaian suatu masalah. Algoritma sendiri ialah kumpulan langkah langkah untuk menyelesaikan suatu masalah. Fase ini merupakan fase paling penting dalam pembuatan suatu program

4. Coding & Documentation

Coding ialah proses penerjemahan algoritma kita ke bahasa pemrograman yang dipahami oleh komputer. Fase ini ialah Proses inti dalam Pemrograman

Dokumentasi sendiri ialah proses penyimpanan suatu algoritma atau coding dalam suatu berkas agar bisa ditinjau di kemudian hari.

5. Testing

Testing ialah proses pengujian apakah suatu program tersebut dapat menyelesaikan masalah kita. Dengan melihat input yang kita masukkan dan output yang kita harapkan

6. Maintenance

Maintenance ialah proses perbaikan jika program tersebut mengalami error dan hal hal yang tidak kita harapkan. Proses Maintenance akan membawa kita kembali ke desain dan pengembangan algoritma
