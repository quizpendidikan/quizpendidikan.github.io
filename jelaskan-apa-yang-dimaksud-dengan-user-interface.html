KETERANGAN :

Quick Access,

kumpulan tombol perintah yang dapat mempercepat kerja anda. Isinya hampir sama dengan perintah-perintah di Office Button.

Title Bar,

nama file dan program aplikasi yang sedang aktif.

Tab Menu Ribbon,

barisan menu yang terdiri dari perintah menu utama Microsoft WORD yaitu Home, Insert, Page Layout, References, Mailing, Review, View.

Ribbon,

barisan tombol-tombol bergambar yang memiliki fungsi tertentu dan digunakan untuk menjalankan suatu perintah dengan cepat dan mudah.

Kontrol Jendela,

kumpulan perintah yang dapat digunakan untuk mengatur ukuran jendela kerja, Minimize, Maximize, Restore Down, dan Close.

Horisontal Ruler,

penggaris yang melintang diatas dokumen kerja yang sedang aktif. yang nantinya dapat kita gunakan untuk membuat ukuran lebar kertas, spasi, tab dan lain sebagainya.

Vertical Ruler,

penggaris yang memanjang kebawah dokumen kerja yang sedang aktif. yang nantinya dapat kita gunakan untuk membuat ukuran panjang kertas, dan spasi baris.

Tombol Viewer,

kumpulan tombol yang berisi perintah tampilan area print dokumen kerja seperti, Print Layout, Full Screen Reading, Web Layout, Outline & Draft.

Zoom,

perintah untuk memperbesar & memperkecil tampilan pada dokumen kerja anda.

Status Bar,

kumpulan informasi dan perintah tentang jumlah page/halaman, Words/kata, dan Langguage/Format Bahasa dari dokumen kerja anda.

Horizontal Scroll Bar,

perintah untuk menggeser layar ke kiri atau ke kanan.

Vertical Scroll Bar,

perintah untuk menggeser layar ke atas atau ke bawah.

18. 1. Sebutkan ada berapa macam user interface, dan berilah contoh masing-masing user interface tersebut.​


Jawaban:

Ada beberapa macam antarmuka pengguna (user interface/UI), dan berikut adalah beberapa contoh:

1. Antarmuka Pengguna Grafis (GUI):

Contoh: Windows, macOS, dan banyak sistem operasi modern menggunakan GUI yang memungkinkan pengguna berinteraksi dengan elemen visual seperti ikon, tombol, dan jendela.

2. Antarmuka Pengguna Command-Line (CLI):

Contoh: Command Prompt di Windows, Terminal di macOS/Linux. Pengguna berinteraksi dengan sistem melalui baris perintah atau sintaksis teks untuk menjalankan perintah.

3. Antarmuka Pengguna Berbasis Teks:

Contoh: Pengedit teks seperti Notepad atau Vim. Pengguna berinteraksi langsung dengan teks untuk membuat atau mengedit informasi.

4. Antarmuka Pengguna Berbasis Web (Web UI):

Contoh: Situs web dan aplikasi web. Pengguna berinteraksi dengan elemen antarmuka yang ditampilkan di browser web.

5. Antarmuka Pengguna Berbasis Suara (Voice User Interface/VUI):

Contoh: Asisten suara seperti Siri, Google Assistant, atau Alexa. Pengguna berkomunikasi dengan sistem melalui perintah suara.

6. Antarmuka Pengguna Berbasis Sentuhan (Touch User Interface/TUI):

Contoh: Layar sentuh pada perangkat mobile seperti smartphone atau tablet. Pengguna berinteraksi dengan menyentuh layar untuk melakukan berbagai tindakan.

7. Antarmuka Pengguna Berbasis Realitas Virtual (Virtual Reality User Interface/VR UI):

Contoh: Pengalaman virtual reality di headset VR. Pengguna berinteraksi dengan dunia virtual menggunakan gerakan atau kontroler khusus.

8. Antarmuka Pengguna Berbasis Gerak (Gesture User Interface):

Contoh: Microsoft Kinect atau teknologi gestur pada perangkat mobile. Pengguna berinteraksi dengan perangkat melalui gerakan fisik atau gestur tertentu.

Setiap jenis antarmuka pengguna memiliki karakteristik unik dan memenuhi kebutuhan berbagai konteks penggunaan.

28. Jelaskan prosedur pembuatan menu pada user interface/ antar muka


Jawaban:

Menerapkan pembuatan antar muka (User Intreface) pada aplikasi Definisi dan Karakteristik Antar Muka ( Interface) Prinsip Perancangan Antar Muka User Interaction Penggunaan Warna pada desain Interface User Guidance

Definisi dan Karakteristik Antar Muka ( Interface) Interface (antar muka) adalah sebuah media yang dapat menjembatani antara user dan sistem untuk melakukan komunikasi. Antar muka mempunyai dua (2) tugas pokok: 1. menterjemahkan semua aksi yang diberikan user sehingga dimengerti system 2. menampilkan hasil operasi dari sistem kedalam bentuk yang dimengerti oleh user. Antar muka banyak sekali kita jumpai di kehidupan tidak hanya dalam software. Contoh interface : tombol start dan Window Explorer dalam OS Windows, remote TV, panel kontrol listrik, dan tombol on/of lampu.

Definisi dan Karakteristik Antar Muka ( Interface) Keuntungan GUI (Graphical User Interface): 1. Mudah dipelajari oleh pengguna yang cukup minim pengalaman dalam menggunakan computer 2. Memungkinkan user untuk dapat berpindah dari satu layar ke layar yang lain tanpa kehilangan informasi yang dibutuhkan. 3. Dapat melakukan akses penuh pada layar untuk menjalankan beberapa macam tugas/keperluan dalam waktu yang singkat.

Prinsip Perancangan Antar Muka 1. User Compatibility kesesuaian tampilan dengan tipikal dari user, karena berbeda user bisa jadi kebutuhan tampilannya berbeda. Misalnya, jika aplikasi diperuntukkan bagi anak-anak, maka jangan menggunakan istilah atau tampilan yang biasa digunakan oleh orang dewasa

Prinsip Perancangan Antar Muka 2. Product Compatibility produk aplikasi yang dihasilkan juga harus sesuai, memiliki tampilan yang sama/serupa, baik untuk user yang awam maupun yang ahli. 3. Task Compatibility fungsional dari task/tugas yang ada harus sesuai dengan tampilannya. Misal untuk pilihan report, orang akan langsung mengartikan akan ada ditampilkan laporan. 30. apa saja referensi user interface?​


1) Nielsen Norman Group (NN/g): NN/g adalah perusahaan konsultan desain pengalaman pengguna yang memiliki banyak artikel, buku, dan panduan praktis tentang desain antarmuka pengguna yang efektif. Mereka sering dianggap sebagai otoritas dalam bidang desain UX dan UI.

2) Material Design: Material Design adalah panduan desain antarmuka pengguna yang dikembangkan oleh Google. Panduan ini memberikan prinsip-prinsip desain yang fokus pada tampilan dan interaksi yang responsif, aliran visual yang mulus, dan penggunaan tampilan material yang realistis.

3) Apple Human Interface Guidelines: Panduan ini diterbitkan oleh Apple dan memberikan pedoman tentang desain antarmuka pengguna untuk aplikasi dan platform Apple seperti iOS, macOS, dan watchOS. Panduan ini menekankan kekonsistenan, kejelasan, dan kegunaan dalam desain antarmuka pengguna.

4) Microsoft Fluent Design System: Fluent Design System adalah panduan desain yang dikembangkan oleh Microsoft untuk platform Windows. Panduan ini mencakup prinsip-prinsip desain seperti kejelasan, kedalaman, gerak alami, dan responsif yang diterapkan dalam desain antarmuka pengguna.

5) IBM Design Language: IBM Design Language adalah panduan desain yang dikembangkan oleh IBM untuk desain antarmuka pengguna pada produk dan layanan mereka. Panduan ini berfokus pada kegunaan, konsistensi, dan kemampuan adaptasi pada berbagai platform.

Selain itu, ada juga banyak buku, artikel, blog, dan sumber daya online lainnya yang membahas desain antarmuka pengguna. Penting untuk mencari referensi dari berbagai sumber untuk memperoleh wawasan yang komprehensif dan dapat diterapkan dalam desain UI yang efektif

36. sebutkan kelebihan mobile user interface dan kekurangan form based interface, touch user interface conversational user interface dan mobile interface​


**Mobile User Interface (UI):**

- *Kelebihan*:

 1. Portabilitas: Mobile UI memungkinkan pengguna untuk mengakses informasi dan layanan kapan saja dan di mana saja melalui perangkat seluler.

 2. Fitur Khusus: Mobile UI dirancang khusus untuk memanfaatkan fitur-fitur perangkat seluler seperti GPS, sensor, dan kamera.

 3. Pengalaman Pengguna yang Lebih Personal: Mobile UI memungkinkan pengguna untuk mengakses aplikasi dan konten yang lebih sesuai dengan preferensi dan lokasi mereka.

- *Kekurangan*:

 1. Ruang Terbatas: Layar perangkat seluler terbatas dalam ukuran, yang dapat membatasi tampilan dan fungsi yang tersedia.

 2. Tantangan Navigasi: Terkadang navigasi dalam aplikasi mobile dapat menjadi rumit, terutama jika tidak didesain dengan baik.

 3. Penggunaan Daya Baterai: Aplikasi mobile dapat mengonsumsi daya baterai yang cukup tinggi, terutama jika digunakan intensif.

**Form-Based Interface:**

- *Kelebihan*:

 1. Struktur yang Jelas: Form-based UI menyediakan struktur yang jelas dan terorganisasi untuk mengumpulkan data dari pengguna.

 2. Konsistensi: Formulir umumnya konsisten dalam tampilan dan penggunaan, yang membuatnya mudah dipahami oleh pengguna.

 3. Validasi Data: Formulir dapat menyediakan validasi data otomatis untuk memastikan data yang diinput benar.

- *Kekurangan*:

 1. Kepentingan Penggunaan Data: Tergantung pada jumlah informasi yang harus dimasukkan, pengisian formulir bisa menjadi tugas yang memakan waktu.

 2. Terkadang Terlalu Formal: Formulir mungkin tidak sesuai untuk situasi yang lebih santai atau non-transaksional.

 3. Potensi Kesalahan Manusia: Kesalahan pengisian data oleh pengguna dapat terjadi.

**Touch User Interface:**

- *Kelebihan*:

 1. Intuitif: Tampilan sentuh (touchscreen) umumnya dianggap intuitif dan mudah digunakan.

 2. Responsif: Antarmuka sentuh umumnya memberikan umpan balik langsung saat pengguna menyentuh layar.

 3. Interaktif: Menggunakan gerakan jari dan gestur untuk mengoperasikan perangkat menyediakan pengalaman interaktif.

- *Kekurangan*:

 1. Dapat Sensitif terhadap Kotoran: Layar sentuh rentan terhadap noda, sidik jari, dan goresan.

 2. Memerlukan Pembiasaan: Pengguna mungkin perlu waktu untuk membiasakan diri dengan gestur dan operasi layar sentuh.

 3. Tidak Cocok untuk Semua Aplikasi: Layar sentuh mungkin tidak cocok untuk semua jenis aplikasi, terutama yang memerlukan pengaturan yang lebih rumit.

**Conversational User Interface (CUI):**

- *Kelebihan*:

 1. Kemudahan Penggunaan: CUI memungkinkan pengguna berinteraksi dengan perangkat atau aplikasi dengan cara yang lebih alami melalui percakapan.

 2. Pengenalan Suara: Pengenalan suara dalam CUI memungkinkan akses cepat ke informasi atau perintah tanpa mengetik.

 3. Akses Hands-Free: CUI memungkinkan pengguna untuk menggunakan perangkat atau aplikasi tanpa harus menggunakan tangan, yang bermanfaat dalam situasi-situasi tertentu seperti saat mengemudi.

- *Kekurangan*:

 1. Terbatas pada Pemahaman: CUI mungkin tidak selalu memahami bahasa atau perintah pengguna dengan baik, terutama dalam konteks yang lebih kompleks.

 2. Terbatas pada Aplikasi Tertentu: CUI mungkin lebih cocok untuk aplikasi yang melibatkan komunikasi lisan, dan tidak semua aplikasi dapat diintegrasikan dengan CUI.

 3. Memerlukan Infrastruktur yang Sesuai: CUI memerlukan infrastruktur pengenalan suara yang sesuai dan koneksi internet yang stabil.
