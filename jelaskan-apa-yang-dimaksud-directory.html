11. jelaskan struktur file dan directory pada sistem operasi Linux


- /bin: berisi user command binnary files.
- /boot: berisi static file untuk proses boot loader
- /dev: berisi file ttg device hardware
- /etc: berisi file untuk konfigurasi system
- /home: berisi area kerja setiap user (kecuali super user/user root ya)
- /lib: berisi file library yg esensial dan sbg kernel modul
- /mnt: berisi mounting point dan bersifat temporer file system
- /media: berisi mounting point khusus untuk removable media device
- /opt: berisi aplikasi add on/tambahan/optional dari berbagai paket software
- /proc: berisi virtual file system untuk akses informasi kernel yg bersangkutan
- /root: berisi area kerja khusus untuk super user (cuma ada di os linux krn sifatnya open source jadi os nya hackable dan bs di customize).
- /sbin: berisi binnary files tapi cuma bisa diakses hirarki user sekelas super user.
- /tmp: berisi file ysng temporer
- /usr: direktori utama setelah main direktori (hirarki file lvl 2) yg berisi file2 aplikasi dan source kernel
- /var: berisi file log system, spool, dan sejenisnya.

18. jelaskan struktur file dan directory pada sistem operasi Linux


OPERASI FILE DAN STRUKTUR DIREKTORI LINUXOPRASI FILE DAN STRUKTUR DIREKTORI LINUX

Pokok Bahasan :

- Struktur Direktory pada Sistem Operasi Linux

Tujuan Belajar
Setelah mempelajari materi dalam bab ini, mahasiswa diharapkan mampu :
- Memahami organisasi file dan direktory pada sistem operasi Linux
- Menciptakan dan manipulasi direktory

DASAR TEORI :
1. ORGANISASI FILE

Sistem file pada Linux menyerupai pepohonan (tree), yaitu dimulai dari root, kemudian direktory dan sub direktory. Sistem file pada Linux diatur secara hierarkikal, yaitu dimulai dari root dengan symbol “/” seperti Gambar 3.1

Kita dapat menciptakan File dan Direktori mulai dari root ke bawah. Direktori adalah file khusus, yang berisi nama file dan INODE (Pointer yang menunjuk ke data / isi file tersebut). Secara logika, direktori dapat berisi File dan Direktori lagi (disebut juga Subdirektori).



Struktur DirektoriStruktur direktori di Unix dan Linux adalah sebuah struktur direktori terpadu di mana semua direktori bersatu di bawah direktori “/” filesystem root. Terlepas dari mana filesystem secara fisik dipasang, semua direktori yang disusun secara hirarki dibawah filesystem root. Struktur Direktori Linux mengikuti “Filesystem Hierarchy Structure (FHS)” dipelihara oleh Free Standars Group meskipun sebagian besar distribusi kadang-kadang cenderung menyimpang dari standar.Mari kita lihat sekilas direktori apa saja yang berada di bawah hirarki filesystem linux.

“/” root
Struktur direktori diawali dengan filesystem root “/” dan memang direktori akar dari seluruh struktur. Partisi dimana / (direktori root) akan ditempatkan pada sistem Unix atau kompatibel dengan Unix.

/boot
Direktori /boot berisi file boot loader termasuk Grub dan LiLO, kernel, initrd, config dan system.map.

/sys
Direktori ini berisi kernel, firmware dan file yang berhubungan dengan sistem.

/sbin
Berisi binari sistem esensial dan peralatan sistem administrasi penting bagi sistem operasi dan kinerjanya.

/bin
Berisi binari penting bagi pengguna dan utilitasnya yang diperlukan dalam mode pengguna sendiri.

/lib
Berisi file-file pustaka untuk semua binari yang berada di dalam direktori /sbin dan /bin./dev
Berisi file sistem esensial dan driver-driver./etc
Direktori ini berisi file konfigurasi sistem esensial termasuk /etc/hosts, /etc/resolve.conf, nsswitch.conf dan file konfigurasi jaringan./home
Semua direktori rumah dari pengguna berada di dalam direktori ini dengan pengecualian direktori rumah akun root yang mana disimpan di dalam direktori /root. Direktori ini berisi file pengguna, pengaturan pengguna, profil dan lain-lain./media
Poin mount untuk media penyimpanan bergerak seperti CD-ROM, USB, Floppy dan lain-lain./mnt
Poin mount generik untuk filesystem sementara. Ini sangat berguna khususnya ketika mengalami permasalahan dan harus menggunakan LiveCD yang di mana anda mungkin harus melakukanmount terhadap filesystem root dan mengubah pengaturan./opt
Sebuah direktori yang jarang sekali digunakan di Linux dan biasanya dipakai untuk perangkat lunak dari pihak ketiga dan bukan merupakan utilitas dari Proyek GNU seperti aplikasi java ataupun virtualbox./usr
Sebuah sub-hirarki terhadap filesystem root yang mana merupakan direktori data pengguna. Berisi file yang merupakan utilitas dan aplikasi tertentu. Terdapat juga direktori dokumentasi dari aplikasi tersebut. /var
Direktori ini biasa di-mount sebagai filesystem pada partisi yang terpisah di bawah root di mana semua konten (isi) variabel seperti logs, file spool untuk printer, crontab, mail, proses yang dilakukan, file lock dan lain-lain. Penting sekali untuk memperhatikan dalam perencanaan pembuatan darifilesystem ini dan perawatannya karena filesystem ini dapat terisi penuh secara cepat dan ketikafilesystem kapasitasnya penuh maka dapat menyebabkan permasalahan pada operasional dari sistem dan aplikasi./tmp
Sebuah filesystem sementara yang menyimpan file-file sementara dan akan dihapus ketika sistem di-reboot (dijalankan kembali). Ada juga sebuah direktori /tmp di dalam direktori /var yang berfungsi sama untuk menyimpan file-file sementara. Salah satu perbedaan di antara keduanya adalah direktori /var/tmp tetap menyimpan file-file di dalamnya dan melindungi ketika sistem reboot. Dengan kata lain, file di dalam /var/tmp tidak akan dibuang setelah reboot.Struktur Direktori secara Visual

















Catatan :
Struktur direktori ini mungkin berbeda dengan distibusi Linux yang lain dan struktur direktori ini sifatnya umum. Struktur direktori di Linux terkadang membingungkan bagi pengguna Linux baru. Sebagian besar distribusi Linux memiliki struktur direktori yang umumnya sama dari struktur direktori dasarnya./ -> root directory
/var -> variabel directory
/home -> user directory
swap -> virtual memory
/boot -> boot directory

30. apa yang di maksud denganmenu pulldown,menu pop-up,menu utama,menu cascading dan menu directory?


Kelas: VIII
Mata Pelajaran: Teknologi Informasi
Materi: Antarmuka Pengguna
Kata Kunci: menu

Pembahasan:

1. pulldown menu

Menu pulldown adalah tambahan yang muncul di bagian bawah ketika kita menekan salah satu tombol menu utama. Menu-menu dalam menu pulldown umumnya berkaitan satu sama lain dan dengan menu utama tadi.

Misalnya bila kita menekan "file" pada berbagai aplikasi, maka akan ada menu puldown dengan pilihan "open" untuk membuka file, "close" untuk menutup file, "save" untuk menyimpan file,  "save as" untuk menyimpan file sebagai file baru, dan "exit" untuk menutup aplikasi.

2. menu pop-up

Menu yang muncul dalam bentuk pop up saat kita melakukan aksi lain. Popup banyak digunakan untuk menampilkan konfirmasi akan tindakan yang kita lakukan sebelumnya, apakah benar akan dilakukan atau tidak.

Misalnya, ketika akan menghapus suatu berkas (file) kita akan mendapatkan menu popup konfirmasi apakah benar akan menghapus atau tidak. Bila kita jawab ya/yes pada menu makan file baru akan terhapus.

3. menu utama

Menu utama adalah menu yang selalu muncul pada aplikasi. Menu utama umumnya bila ditekan akan memunculkan menu lain dalam bentuk pulldown.

Misalnya pada aplikasi pengolah dokumen seperti Microsoft Wor akan ada menu utama "File", "Edit", "Layout", "Reference", "Insert", dan sebagainya. Setiap menu bila ditekan akan memunculkan menu lain.

4. Menu cascading

Menu cascading adalah menu yang muncul di bagian samping dari menu sebelumnya. Menu ini bisa bersusun beberapa lapis atau tingkatan bagian.

Misalnya pada menu-menu pada tombol Windows, bila kita klik dapat memmunculkan pilihan lebih lanjut.

5. menu directory

Menu directory adalah menu yang muncul ketika mengklik kanan satu atau beberapa berkas atau ketika memilih beberapa teks.

Misalnya bila kita klik kaknan suatu file akan muncul pilihan "open" untuk membuka file, "renane" untuk mengganti nama file,, "delete" untuk menghapus file, dan sebagainya

31. Jelaskan perbedaan antara single level directory, two level directory dan hierarchical


Jenis struktur direktori dalam sistem file:

Single level directory. Struktur direktori ini merupakan struktur direktori yang paling sederhana. Semua file disimpan dalam direktori yang sama. Two level directory (direktori dua tingkat). Direktori dua tingkat membuat direktori  terpisah untuk setiap pengguna, yang disebut Direktori File Pengguna (UFD). Saat pengguna masuk, root file dipanggil.Hierarchical merupakan suatu sistem Debian yang diorganisasikan mengikuti Filesystem Hierarchy Standard (FHS). Standar ini mendefinisikan tujuan masing-masing direktori.

Pembahasan

Beberapa sistem komputer menyimpan banyak file di disk, jadi Anda memerlukan struktur untuk mengatur data Anda untuk pengelolaan yang lebih mudah. Silberschatz, Galvin, dan Gagne mengklasifikasikan operasi pada direktori sebagai berikut:

Cari filenya. Menemukan entri untuk file tertentu dalam struktur direktori. File dengan nama  simbolik dan nama yang mirip menunjukkan koneksi antar file. Buat file. Anda perlu membuat file baru  dan menambahkannya ke  direktori.  Hapus file. Ketika Anda tidak lagi membutuhkan file, Anda harus menghapusnya dari direktori.  Menampilkan isi direktori. Menampilkan daftar file dalam direktori dan semua isi direktori dari file dalam daftar itu. Silahkan ganti nama filenya. Nama file mencerminkan isi file kepada pengguna. Oleh karena itu, nama file harus dapat diubah jika isi dan penggunaannya berubah atau menjadi tidak sesuai. Mengganti nama file dapat mengubah lokasinya dalam struktur direktori. Akses ke sistem file. Akses semua direktori dan file dalam struktur direktori. Pada titik tertentu, sangat disarankan untuk menyimpan konten dan struktur seluruh sistem file.
